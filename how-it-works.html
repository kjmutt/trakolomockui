How it works — plain English
The same architecture as Application architecture and Cloud hosting, explained with an office-building analogy for anyone who doesn't live in infrastructure diagrams day to day.
The big picture
Think of Trakolo as one office building shared by many different companies (tenants). Most rent space on a shared floor. One or two demand — and pay for — their own private suite. Everything below is a room, a desk, or a job title in that building, mapped to the real Azure service that plays that role.
The front door
Azure DNS + Front Door
Every company's employees type something like acme.trakolo.com. Azure DNS is just the building directory saying "this address exists, go this way." Front Door is the receptionist at the entrance — checks ID (that's the padlock icon / HTTPS, so nobody can eavesdrop) and points each visitor to the right floor. One receptionist desk serves everybody; nobody gets their own private entrance.
The office space
Azure Static Web Apps + Container Apps
Static Web Apps hosts the part employees actually see — the buttons, screens, forms. Think of it as the lobby signage and public counters. Container Apps runs the staff actually doing the work behind the counter — creating tickets, updating assets, running reports. It's one team, not four separate departments, because the modules constantly hand paperwork to each other (a change request literally references a dev backlog item and back) — splitting them into separate departments would mean memos between buildings instead of turning around and asking a coworker.

Almost every company shares the same open-plan floor — cheaper, and Azure just adds more desks when it gets busy. One or two companies with strict contracts can pay for a locked private office instead (that's the compute_pool setting) — same building, same trained staff, just a different room.
The front desk directory
Azure Cache for Redis
Before staff can help a visitor, they need to know: which company is this, and which filing cabinet belongs to them? Azure Cache for Redis is the front desk's quick-reference card — instead of walking to records every single time, the desk just glances at a cheat sheet it already has memorized. Fast, so nobody waits.
The filing room
Azure Database for PostgreSQL + Blob Storage
This is where everyone's actual data — tickets, assets, documents — physically lives. Most companies share one big filing cabinet system (a shared Postgres database), but every drawer is labeled with the company's name (tenant_id) so nobody can open someone else's folder. A few big customers get their own separate locked filing room entirely — a dedicated database just for them, because their contract requires it. There's also a photocopy-only room (a read replica), so someone pulling a big report doesn't hold up people actively filing new paperwork. Large attachments go to Blob Storage — an off-site warehouse rather than a desk drawer, cheaper for bulky stuff.
The mailroom — five clerks, not one
Azure Container Apps jobs + Service Bus
Some jobs don't need to happen the instant someone clicks a button — reminders, deadline checks, webhook deliveries, weekly reports. Instead of one clerk doing all four jobs, there are five separate clerks, each with one:
watches deadlines — worker-sla applies business rules — worker-rules delivers webhooks — worker-webhooks builds reports — worker-reports runs the AI assistant — worker-ai-agent

If the webhook clerk calls in sick (crashes), the deadline clerk still shows up. In the old design there was one clerk doing everything — if that person had a bad day, every company's mail stopped moving at once. That actually happened before, which is why it's split now.
The security office
Azure Monitor + Application Insights → PagerDuty
Cameras and alarms watching the whole building. Azure Monitor + Application Insights are the cameras — they see everything happening. If something looks wrong, it triggers an alarm that goes straight to PagerDuty, which calls a real human on-call engineer — like a security alarm calling the guard's cellphone at 2am, instead of just blinking a light nobody's watching.
The safe
Azure Key Vault
Every building has one locked safe holding the master keys — database passwords, the private key used to "sign" a customer's license file so it can't be forged. Key Vault is that safe. Nobody writes the combination on a sticky note (a plaintext .env file) in a real production building.

The one-sentence version
Everybody shares one well-staffed office building and one shared filing system, a couple of big customers get their own locked room, five separate mailroom clerks handle background work so one clerk's bad day doesn't stop everyone's mail, and cameras + an alarm system make sure someone gets called the moment anything breaks.
