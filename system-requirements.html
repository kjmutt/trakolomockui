System requirements
Service versions and hardware sizing for the standalone/on-premise install (install/docker-compose.yml) — current as of July 2026, re-check the source links before every release.

In one line
Postgres 16 (pinned) is fine through 2028 but Postgres 18 is the current stable if starting fresh; Redis should move from the 7.x pin to 8.x; nginx's floating alpine tag needs no action. Hardware scales with concurrent users, not company size — Postgres is the resource-hungry piece.

Hardware sizing — single-host on-prem install
This is the perpetual-license edition — one host, one docker compose up, no Kubernetes — so sizing is per customer, not the 10,000-concurrent multi-tenant SaaS scale in Cloud hosting. Postgres is the RAM-hungry piece; disk should be SSD/NVMe specifically because ticket search and Dev-module diff/blob operations are I/O-heavy.
